# 커밋 메시지 유형

-  `feat`: 새로운 기능 추가
-  `fix`: 버그 수정
-  `BREAKING CHANGE`: 호환되지 않는 변경 사항
-  `docs`: 문서 변경 사항
-  `style`: 스타일 및 서식 변경 사항
-  `ci`: CI/CD 관련 변경 사항
-  `refactor`: 코드 품질 개선 변경 사항
-  `test`: 테스트 코드 변경 사항
-  `build`: 빌드 시스템 변경 사항
-  `perf`: 성능 개선 변경 사항
-  `chore`: 기타 변경 사항
-  `delete`: remove와 유사하게, 주로 파일이나 코드의 삭제.
